2Ô∏è‚É£ Basic Usage - Auto-Routing

Let the router intelligently classify and route tasks:

import requests

# Process a task with automatic classification
response = requests.post(
    "http://localhost:8000/task",
    json={
        "prompt": "Write a Python function to sort a list",
        "task_type": None,  # Auto-classify
        "stream": False
    }
)

result = response.json()
print(f"Model used: {result['model_used']}")
print(f"Classification: {result['task_classification']}")
print(f"Result: {result['result']}")

5Ô∏è‚É£ Classify Without Executing

Preview classification before processing:

    "http://localhost:8000/classify",
    json={"prompt": "Write a Python function"}
)

classification = response.json()
print(f"Classification: {classification['classification']}")
print(f"Recommended Model: {classification['recommended_model']}")
print(f"Available Models: {classification['available_models_in_category']}")

6Ô∏è‚É£ Error Handling

Handle errors gracefully:

try:
    result = client.process("Your prompt here")
    print(f"Success: {result['result']}")

except requests.exceptions.ConnectionError:
    print("Error: Cannot connect to router at http://localhost:8000")
    print("Make sure the router is running: python ollama_router.py")

except Exception as e:
    print(f"Error: {type(e).__name__}: {e}")

7Ô∏è‚É£ Batch Processing

Process multiple tasks efficiently:

tasks = [
    "Write a Python function",
    "Analyze this image",
    "Explain quantum computing"
]

results = []
for task in tasks:
    result = client.process(task)
    results.append({
        'task': task,
        'model': result['model_used'],
        'classification': result['task_classification']
    })
    print(f"‚úì {result['task_classification']}: {result['model_used']}")

# Analyze distribution
from collections import Counter
models_used = Counter(r['model'] for r in results)
print(f"\nModel usage: {dict(models_used)}")

üí° Best Practices

  • Let it auto-classify: The router's intelligence shines when you let it classify tasks
  • Cache results: Avoid processing identical prompts multiple times
  • Check connectivity: Start with curl http://localhost:8000/health
  • Monitor usage: Check the Dashboard tab for performance insights
  • Handle timeouts: Complex tasks might take 5+ minutes (default timeout)
  • Batch wisely: Process related tasks together for efficiency

üîß Troubleshooting

‚ùå "Cannot connect to Ollama"

Make sure Ollama is running:

ollama serve
Then check:
curl http://localhost:11434/api/tags

‚ùå "No models available"

Pull a model in Ollama:

Then restart the router

‚ùå "Router not responding"

Verify it's running:

curl http://localhost:8000/health
Or restart:
python ollama_router.py

‚è±Ô∏è Limits & Timeouts

  • Request Timeout: 300 seconds (5 minutes) for task execution
  • Classification Timeout: 30 seconds
  • Max Concurrent Requests: Limited by system resources
  • Model Loading Time: First request to a model may take 10-30 seconds
  • Recommendation: For production, use load balancing and caching
